# Dagua Maintenance Checklist

This is the central checklist of surfaces that need regular updates as Dagua evolves.
It is intentionally broader than a release checklist: it covers code-adjacent artifacts,
generated docs, benchmark outputs, showcase assets, notebooks, and public explanations.

Use it when preparing a push, a release, a major feature branch, or a documentation refresh.

## Core Public API

- `dagua/__init__.py`
  - public re-exports
  - `draw()` convenience behavior
  - animation / tour / poster exports
- `dagua/graph.py`
  - graph lifecycle behavior
  - cached layout state
  - graph creation entrypoints
- `dagua/config.py`
  - `LayoutConfig`
  - optimization hyperparameters
  - direction / device / spacing defaults
- `dagua/styles.py`
  - `NodeStyle`, `EdgeStyle`, `ClusterStyle`, `GraphStyle`
  - built-in themes
  - theme registry
- `dagua/defaults.py`
  - global default configuration surface

Update when:
- public functions are added, removed, renamed, or re-scoped
- defaults change
- lifecycle behavior changes
- a new built-in theme is added
- a new style/config field is added

## Practical Push Checklist

Before pushing a meaningful user-facing change, ask:

1. Did the public API or defaults change?
2. Did any generated docs become stale?
3. Did any showcase or explainer visuals become stale?
4. Did the notebooks stop representing the best workflow?
5. Did the benchmark/report interpretation change?
6. Did the CLI examples or README become inaccurate?
7. Did any agent-facing guidance become stale?
8. Did any eval graph corpora need new challenge cases?
9. Did any cross-repo integration surface drift?

If the answer is yes to any of these, update the corresponding surfaces before pushing.
