# Dagua Conventions

## Naming

- **Modules**: lowercase, no underscores except multi-word (`init_placement.py`, `edge_optimization.py`)
- **Classes**: PascalCase. Core types: `DaguaGraph`, `LayoutConfig`, `NodeStyle`, `EdgeStyle`, `ClusterStyle`, `Theme`, `Flex`, `LayoutFlex`, `AlignGroup`
- **Functions**: snake_case. Public API uses short verbs: `draw()`, `layout()`, `render()`, `configure()`
- **Constants**: UPPER_SNAKE_CASE. Themes: `DEFAULT_THEME`, `DARK_THEME`, `MINIMAL_THEME`, `TORCHLENS_THEME`
- **Test files**: `test_<module>.py`, mirroring source layout
- **YAML graphs**: kebab-case (`neural_net.yaml`, `data_pipeline.yaml`)
- **Task IDs**: kebab-case (`fix-trace-module`, `add-sweep-dataclass`)

## Error Handling

- Errors are raised, never swallowed. No silent failures.
- `ValueError` for invalid user input (bad node IDs, invalid configs).
- `ImportError` with helpful message for missing optional deps (matplotlib, pyyaml, igraph).
- Layout engine failures should surface as clear errors, not NaN positions.

## Testing Patterns

- **Fixtures**: shared sample graphs in `tests/conftest.py` (e.g., diamond graph, pipeline graph)
- **Layout tests**: check convergence properties (e.g., DAG constraint reduces y-violations),
  not exact coordinates — optimization is stochastic
- **Render tests**: check structural output (SVG element count, figure created),
  not pixel-perfect comparison
- **Markers**: `smoke` (<10s total), `slow` (>10s each), `gpu` (requires CUDA),
  `benchmark` (eval suite), `rare` (5M+ nodes)
- **Regression tests**: when fixing a regression, add the narrowest test that would have caught it
- **Smoke tests**: preferred place to pin workflow and scale regressions in miniature

## Documentation

- **Docstring format**: NumPy style
- **Public/orchestration-heavy functions**: explicit docstrings required
- **Module docstrings**: explain role and boundaries, not just restate the filename
- **Comments**: prefer a few strong section comments over many shallow ones
- **Type annotations**: improve in orchestration modules rather than letting `Any` spread
- **Generated docs**: glossary (`scripts/build_glossary.py`), gallery (`scripts/build_gallery.py`),
  algorithm explainer (`scripts/build_how_dagua_works.py`)

## Git

- **Commit messages**: conventional commits — `fix(scope):`, `feat(scope):`, `chore(scope):`, etc.
- **Branch naming**: `feat/<description>`, `fix/<description>`, `codex/<task-id>` for dispatched tasks
- **Main branch**: `main` — protected, releases triggered on push
- **Release**: semantic-release v9, `fix:` → patch, `feat:` → minor, `feat!:` → major
- **No AI attribution**: never add Co-Authored-By for Claude/Codex, no "generated by" comments

## Code Style

- **Line length**: 100 (enforced by ruff)
- **Target Python**: 3.9+ (but dev uses 3.11)
- **Formatter**: `ruff format`
- **Linter**: `ruff check --fix` (rules: E, F, W, I)
- **Type checker**: mypy (strict on `dagua/cli.py`, relaxed elsewhere)
- **Dependencies**: PyTorch only required. matplotlib, pyyaml, igraph, scipy, pydot all optional.

## Layout Engine Conventions

- Layout modules (`dagua/layout/`) are **pure torch** — no imports from dagua core.
- Constraints are callables: `(pos: Tensor, graph_data: dict) -> Tensor` (scalar loss).
- Position tensors are always N×2 (x, y). Edge index is always 2×E.
- Bottom-up default direction (matches DNN forward pass).
- Deterministic by default: `seed=42`, `seed=None` to opt out.

## Style System Conventions

- 5-level cascade: per-element > cluster member style > theme type > graph default > global default
- Thread-safe globals via `dagua.configure()` and `dagua.defaults()` context manager
- Themes are immutable named bundles; override via cascade, don't mutate
